Mastering the Command Line
August 8, 2022 · 2 min · Jan Scholz

The command line is as essential to software engineering as a brush to a painter’s. Many tasks can be rapidly executed with a “one-liner” in the terminal, be it the copying of a file to a new directory (cp downloads/file.txt documents/), searching for document (find . -iname 'draft.*'), or sorting a list of names (sort names.txt).

The knowledge of how to glue various utilities together and what arguments to use is referred to as “Command Line Fu”. Here’s is a popular one-liner from commandlinefu.com:

history | awk '{print $2}' | sort | uniq -c | sort -rn | head

This command cleverly uses history, awk, sort, uniq, and head to show the top ten most frequently used commands in your history.

Know the tools#

Being aware of the ‘buffet’ of tools is the first step to master the command line. According to the Unix philosophy “programs should do only one thing and do it well”. Unfortunately that means that there are a lot of programs. Most Unix/Linux-based operating systems come with over 100 text/shell utilities. Plus, developers tend to install additional command line interfaces (CLI), such as Docker, Node, and the gcloud CLI. Image and document processing tools, such as ImageMagik, Ghostscript, and Pandoc can further add to the list.

An experienced developer should be familiar with about 50 tools, their purpose, and their most common options/arguments. The reader can test themselves on this list of tools you should know.

Know when to stop#

Coming up with commands can become a sport. It’s possible to get absorbed into crafting the perfect one-liner1.

Bash is powerful, but it’s geared toward processing of strings and interacting with the file system. If you find yourself building ASCII-based user interfaces or doing numerical calculations with bash, you should probably switch to a more appropriate programming language.

List of tools you should Know#

Alias, awk/sed, cat, cd, chmod, chown, clear, cp, cut, df, echo, find, git, grep, head, history, kill, less/more, ln, ls, man, mkdir, mv, nano/vi, ps, pwd, rev, rsync, shutdown, sort, split, sudo, tail, tar/unzip/zip, tee, top/htop, touch, uname, watch, wget/curl, which, and yes.
